# Camofox Kubernetes Deployment Plan

## Scope

Create a minimal production-ready Kubernetes overlay for running
`jo-inc/camofox-browser` alongside the local Surogates deployment. This
prepares the cluster service dependency needed before wiring browser tools
into the harness and `agent-chat-react`.

## Checklist

1. [x] Add a dedicated kustomize overlay under `k8s/camofox-browser`.
2. [x] Define namespace, service account, persistent storage, deployment, and service manifests.
3. [x] Enable interactive login support with noVNC configuration.
4. [x] Add network policy allowing only Surogates API, worker, and sandbox pods to reach the browser service.
5. [x] Document image build/import, secret creation, apply, health check, and service URL.
6. [x] Validate the overlay with `kubectl --dry-run=client`.

## Deferred

1. [ ] Wire Surogates browser tools to `http://camofox-browser.surogate-default.svc.cluster.local:9377`.
2. [ ] Add an API/frontend proxy path for noVNC so `agent-chat-react` can display interactive login sessions.
3. [ ] Add harness integration tests against a running Camofox service.
